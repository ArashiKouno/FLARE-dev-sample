# エラーハンドリング方針

## フロントエンド（Vue）

- API呼び出し時の失敗（例: ネットワークエラー）は`try-catch`でログ出力

## バックエンド（Express）

- DBエラー → `res.status(500).send(err)`
- バリデーション失敗時の処理は未実装（今後の拡張）

## 想定されるエラー例

| エラー内容               | 対応ステータス | 備考               |
|--------------------------|----------------|--------------------|
| DB接続失敗               | 500            | 起動時ログで通知   |
| データ挿入時の失敗       | 500            | POST処理中         |
